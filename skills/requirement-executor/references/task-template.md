# 任務文件詳細模板

本文件提供任務文件的詳細模板和填寫說明。

## 基礎任務模板

```markdown
# Task-[編號]: [任務標題]

## 對應需求
- REQ-XXX: [需求名稱和簡述]

## 任務目標
[用 1-2 句話明確描述此任務要達成什麼目標]

## 執行步驟
1. [具體可執行的步驟 1]
   - 細節說明
   - 預期結果
2. [具體可執行的步驟 2]
3. [具體可執行的步驟 3]

## 驗收標準
- [ ] 標準 1: [可量化的標準]
- [ ] 標準 2: [可量化的標準]
- [ ] 標準 3: [可量化的標準]

## 預期輸出
- **文件**: 
  - `path/to/output1.ext`
  - `path/to/output2.ext`
- **結果**: [描述預期的結果狀態]

## 依賴任務
- Task-XX: [說明為什麼依賴]
- Task-YY: [說明為什麼依賴]

## 狀態
- [ ] 未開始
- [ ] 進行中
- [ ] 已完成
- [ ] 需要修正

## 執行記錄
[執行時填寫]
```

## 填寫指南

### 任務標題命名規範
- 使用動詞開頭
- 簡潔但描述性
- 範例：
  - ✓ "實作用戶認證 API"
  - ✓ "設計資料庫架構"
  - ✗ "API"（太簡短）
  - ✗ "實作所有功能"（太籠統）

### 驗收標準撰寫原則
- **可測試**: 必須能夠明確判定是否通過
- **可量化**: 盡量使用數字或明確條件
- **獨立**: 每個標準應該獨立評估
- **完整**: 涵蓋所有重要面向

範例：
```markdown
- [ ] API 回應時間 < 200ms (P95)
- [ ] 單元測試覆蓋率 ≥ 80%
- [ ] 所有 API 端點都有 OpenAPI 文檔
- [ ] 代碼通過 linter 檢查（0 errors）
```

### 執行步驟撰寫技巧
- 按時間順序排列
- 每步驟應該清晰可執行
- 包含必要的命令或代碼範例
- 說明預期結果以便驗證

### 狀態更新時機
1. **未開始** → **進行中**: 開始執行第一個步驟時
2. **進行中** → **已完成**: 所有步驟執行完且初步驗證通過
3. **已完成** → **需要修正**: 驗證發現問題
4. **需要修正** → **已完成**: 修正完成並重新驗證通過

## 不同類型任務的模板變化

### 開發類任務
額外包含：
- 代碼規範要求
- 測試要求
- 文檔要求
- Code Review 檢查點

### 文檔類任務
額外包含：
- 文檔結構要求
- 內容完整性檢查
- 格式規範
- 審閱流程

### 配置類任務
額外包含：
- 配置項目清單
- 環境差異說明
- 驗證方法
- 回滾計畫

### 測試類任務
額外包含：
- 測試範圍
- 測試案例清單
- 測試環境要求
- 缺陷追蹤

## 任務連結與追蹤

### 在任務間建立連結
```markdown
## 相關任務
- **前置任務**: [Task-05](05_task_setup_database.md) - 提供資料庫連線
- **後續任務**: [Task-08](08_task_api_integration.md) - 使用本任務的 API
- **並行任務**: [Task-07](07_task_frontend_ui.md) - 可同時進行
```

### 追蹤進度
可在任務文件頂部添加進度條：
```markdown
**進度**: ████████░░ 80%

- [x] 步驟 1
- [x] 步驟 2
- [x] 步驟 3
- [x] 步驟 4
- [ ] 步驟 5
```
