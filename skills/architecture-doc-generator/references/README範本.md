## 使用指引

產生 README.md 時：

1. **替換專案資訊**：
   - 將 `[專案名稱]` 替換為實際專案名稱
   - 更新 Bounded Context 列表
   - 填入實際的技術版本

2. **調整文件列表**：
   - 僅列出實際產生的文件
   - 如果某些文件未產生，從列表中移除

3. **客製化內容**：
   - 根據專案特性調整「快速開始」建議
   - 更新品質屬性目標的實際數值
   - 填入團隊聯絡資訊

4. **更新日期與版本**：
   - 填入當前日期
   - 設定適當的專案版本號
